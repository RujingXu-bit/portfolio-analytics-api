# 项目说明（中文 · 给完全不懂代码的人看）

这份说明从零解释这个项目是什么、怎么启动、怎么使用，以及平时怎么安全地
开关它。你不需要理解代码，按顺序照做即可。

如果某一步看不懂，可以把这一份说明和报错文字发给 AI，但不要发送 `.env`
文件、密码、`access_token`、JWT、API Key 或其他秘密。

## 1. 先用一句话理解它

这是一个“投资组合分析服务”。你可以：

1. 注册和登录；
2. 创建一个投资组合；
3. 记录入金、出金、买入和卖出；
4. 根据历史市场价格查看收益、波动、最大回撤和 Sharpe Ratio；
5. 获得一份历史风险说明。

它是后端服务，不是带漂亮首页的手机 App。启动后，主要通过浏览器里的操作
页面使用：<http://127.0.0.1:8000/docs>。

## 2. 它不能做什么

这个项目不会：

- 预测股票未来涨跌；
- 自动买卖股票；
- 承诺收益；
- 连接你的银行或券商账户；
- 自动换算多种货币；
- 作为生产环境的正式理财系统使用。

它提供的是历史分析和项目演示，不是投资建议。不要把真实账户密码、券商密钥
或重要个人财务数据放进本地演示环境。

## 3. 第一次使用前需要准备什么

需要安装两个工具：

1. Docker Desktop：负责运行数据库和缓存。
   官方说明：<https://docs.docker.com/desktop/>
2. uv：负责安装这个项目需要的软件。
   官方说明：<https://docs.astral.sh/uv/getting-started/installation/>

还需要会打开“终端”：

- macOS：打开“终端”App；
- Windows：打开 PowerShell；
- 如果不知道怎么打开，搜索“如何打开 macOS 终端”或“如何打开 PowerShell”。

本文后面的可直接照抄步骤以当前这台 macOS 电脑为准。Windows 用户不要直接
照抄路径和 `cp` 命令，可以把本说明发给 AI，让它逐条翻译成 PowerShell 命令。
每次只复制一个代码框里的命令，粘贴到终端，然后按回车。

## 4. 第一次安装：只需要做一次

### 第一步：启动 Docker Desktop

打开 Docker Desktop，等到它显示 Docker 已经运行。不要关闭它。

### 第二步：进入项目文件夹

打开终端，输入：

```bash
cd "/Users/delia/Documents/fintech project"
```

如果项目以后被移动到其他位置，请把引号内路径换成新的项目文件夹路径。

### 第三步：检查工具是否可用

```bash
docker --version
```

```bash
uv --version
```

两条命令都应该显示版本号。如果出现 `command not found`，说明对应工具还没有
安装好，请回到上一节的官方说明。

### 第四步：安装项目需要的软件

```bash
make install
```

第一次可能需要几分钟。看到命令正常结束，并重新出现终端输入提示，就可以
继续。

### 第五步：建立本地设置文件

```bash
cp .env.example .env
```

`.env` 是只留在你电脑上的设置文件。它不会被 Git 提交。

### 第六步：设置本地登录密钥

先生成一段随机文字：

```bash
openssl rand -hex 32
```

复制终端输出。然后用普通文本编辑器打开项目里的 `.env` 文件，找到：

```text
JWT_SECRET_KEY=replace_me
```

把 `replace_me` 替换成刚才生成的随机文字并保存。

重要提醒：

- 不要把这段随机文字发给别人或 AI；
- 不要把 `.env` 上传到网盘、聊天或 GitHub；
- `DEEPSEEK_API_KEY` 可以保持空白，项目仍然能生成确定性的风险说明。

## 5. 第一次启动

确认 Docker Desktop 正在运行，并且终端仍在项目文件夹中。

### 第一步：启动数据库和缓存

```bash
make infra-up
```

### 第二步：检查它们是否健康

```bash
make infra-check
```

正常情况下会看到 PostgreSQL 可以接受连接，以及 Redis 返回 `PONG`。

### 第三步：建立数据库表

```bash
make db-upgrade
```

这一步不会清空数据库。它负责把数据库更新到当前项目需要的版本。

### 第四步：启动项目

```bash
make dev
```

看到类似下面的信息说明启动成功：

```text
Uvicorn running on http://127.0.0.1:8000
```

这个终端窗口需要一直开着。不要在这个窗口继续输入其他命令。

### 第五步：打开浏览器

打开：

<http://127.0.0.1:8000/docs>

如果能看到 “Portfolio Analytics API” 和一组接口，说明项目已经可以使用。

## 6. 最简单的完整演示

保持运行项目的终端不动，再打开一个新的终端窗口，进入项目文件夹：

```bash
cd "/Users/delia/Documents/fintech project"
```

然后执行：

```bash
make demo
```

这个命令会自动完成：

1. 创建一个新的演示用户；
2. 登录；
3. 创建投资组合；
4. 记录一笔入金和一笔买入；
5. 再次发送同一笔买入，证明不会重复记账；
6. 获取历史分析；
7. 获取风险说明。

正常结束时会显示一段 JSON 结果。JSON 可以理解为系统整理好的表格数据。

这个演示会通过 yfinance 获取真实历史价格，因此需要网络，但不需要市场数据
API Key。网络或 yfinance 临时不可用时，项目会返回明确错误，不会编造结果。

## 7. 在浏览器里手动操作

如果你只想快速看效果，使用上一节的 `make demo` 就够了。如果想逐步体验，
请打开 <http://127.0.0.1:8000/docs>，按下面顺序操作。

页面中每一个接口都可以点击展开，再点击 **Try it out**，填写内容，最后点击
**Execute**。

### 7.1 注册

找到 `POST /auth/register`，填写：

```json
{
  "email": "my-demo@example.com",
  "password": "my-local-demo-password"
}
```

密码至少需要 12 个字符。这里不要使用你真实邮箱的真实密码。

### 7.2 登录

找到 `POST /auth/login`，填写相同的邮箱和密码。

返回结果里会有 `access_token`。复制它，但不要发给别人。

### 7.3 让后续操作知道你已经登录

点击页面右上角的 **Authorize**，把刚复制的 `access_token` 粘贴进去，然后确认。
通常只粘贴 token 本身，不需要手动加 `Bearer`。

### 7.4 创建投资组合

找到 `POST /portfolios`，填写：

```json
{
  "name": "我的演示组合",
  "base_currency": "USD"
}
```

返回结果里的 `id` 是 Portfolio ID。后面需要把它填入 `portfolio_id`。

### 7.5 先记录入金

找到 `POST /portfolios/{portfolio_id}/transactions`。

把刚才的 Portfolio ID 填入 `portfolio_id`，请求内容填写：

```json
{
  "external_id": "demo-deposit-001",
  "transaction_type": "DEPOSIT",
  "occurred_at": "2026-01-02T08:00:00Z",
  "cash_amount": "1000",
  "fees": "0"
}
```

### 7.6 再记录买入

仍然使用 `POST /portfolios/{portfolio_id}/transactions`：

```json
{
  "external_id": "demo-buy-001",
  "transaction_type": "BUY",
  "occurred_at": "2026-01-02T09:00:00Z",
  "symbol": "AAPL",
  "quantity": "2",
  "unit_price": "100",
  "fees": "1"
}
```

`external_id` 是防止重复记账的编号。同一个 Portfolio 内，每一笔不同交易都应该
使用不同编号。完全相同的请求重复发送时，系统会返回原交易，不会再记一笔。

### 7.7 查看交易记录

使用 `GET /portfolios/{portfolio_id}/transactions`，填写 Portfolio ID 后执行。

### 7.8 查看分析

使用 `GET /portfolios/{portfolio_id}/analytics`：

- `portfolio_id`：你的 Portfolio ID；
- `start_date`：`2026-01-02`；
- `end_date`：`2026-01-30`。

这一步需要网络，因为系统会获取 AAPL 的历史价格。

### 7.9 查看风险说明

使用 `POST /portfolios/{portfolio_id}/insights`，填写同样的 Portfolio ID、
`start_date` 和 `end_date`。

没有设置 DeepSeek Key 时，系统会使用内置规则生成说明，这不是错误。

## 8. 如何看懂分析结果

常见字段可以这样理解：

| 字段 | 通俗解释 |
|---|---|
| `as_of` | 这份结果实际使用数据的最后一天。 |
| `simple_return` | 这段时间的简单收益率；`0.05` 大约是 5%，`-0.05` 大约是 -5%。 |
| `annualized_volatility` | 历史波动大小；越高代表过去价格变化越剧烈，不等于未来一定更危险。 |
| `max_drawdown` | 历史上从高点跌到低点的最大幅度，通常是负数。 |
| `sharpe_ratio` | 用历史超额收益和波动做的比值；它可能为空或为负，也不是未来收益保证。 |
| `portfolio_value` | 最后一个分析日的组合总价值。 |
| `cash_balance` | 组合里剩余的现金。 |
| `asset_weights` | 每个证券占整个组合的比例。 |
| `stale` | `false` 表示使用直接获取或仍有效的缓存；`true` 表示上游失败后使用了明确标记的旧缓存。 |
| `methodology` | 计算口径，例如 adjusted close、252 个年化周期和无风险利率假设。 |

当前 4% 无风险利率是注明日期的演示假设，不是实时利率。所有结果都是历史分析，
不能用来保证未来收益。

## 9. 平时每天怎么开和关

### 开始使用

1. 打开 Docker Desktop；
2. 打开终端并进入项目文件夹；
3. 执行下面三条命令：

```bash
make infra-up
```

```bash
make db-upgrade
```

```bash
make dev
```

4. 打开 <http://127.0.0.1:8000/docs>。

### 结束使用

1. 回到正在运行 `make dev` 的终端；
2. 按 `Control + C` 停止项目；
3. 执行：

```bash
make infra-down
```

这会停止 PostgreSQL 和 Redis，但不会删除 PostgreSQL 数据卷。下次启动后，
已保存的本地数据仍然存在。

不要随便运行 `docker compose down -v`，其中 `-v` 可能删除数据库卷。

## 10. 更新项目文件后怎么做

如果开发者给了你新版项目文件：

1. 先停止正在运行的项目；
2. 在新项目目录执行：

```bash
make install
```

3. 启动 Docker Desktop 和基础设施；
4. 执行：

```bash
make db-upgrade
```

5. 再执行 `make dev`。

不要手工修改数据库表，也不要删除数据库卷来“解决”升级问题。

## 11. 如何自己检查项目是否正常

先启动专门的测试数据库和 Redis：

```bash
make infra-test-up
```

运行快速质量检查：

```bash
make check
```

运行完整单元测试和集成测试：

```bash
make test-all
```

结束后停止测试服务：

```bash
make infra-test-down
```

测试服务与平时使用的开发数据库分开。不要把真实凭据放进测试环境。

## 12. 常见问题

### `docker: command not found`

Docker 没有安装，或安装后还没有重新打开终端。安装并启动 Docker Desktop。

### `Cannot connect to the Docker daemon`

Docker Desktop 没有启动完成。打开它并等到显示正在运行。

### `uv: command not found`

uv 没有安装好。按照 uv 官方安装说明操作，然后重新打开终端。

### `make: command not found`

系统缺少 `make`。macOS 可以安装 Apple Command Line Tools；如果不确定，复制
完整报错问 AI，但不要发送 `.env`。

### `JWT_SECRET_KEY must be configured`

`.env` 不存在，或者 `JWT_SECRET_KEY` 仍为空。重新检查“第一次安装”的第五、
第六步。

### PostgreSQL 或 Redis 连接失败

先确认 Docker Desktop 正在运行，再执行：

```bash
make infra-up
```

```bash
make infra-check
```

### 端口已经被占用

报错里可能出现 `port is already allocated` 或 `address already in use`。这说明
其他程序正在使用 8000、55432 或 6379 等端口。不要随便结束不认识的程序；把
报错发给 AI 或开发者，请对方帮助确认占用者。

### 注册时提示邮箱已存在

换一个演示邮箱，或者直接用原邮箱登录。系统不会把密码告诉你。

### 返回 401

表示没有登录、token 填错或 token 已过期。重新执行登录，再到 **Authorize**
填写新的 `access_token`。

### 返回 404

Portfolio ID 可能填错，或者这个 Portfolio 不属于当前登录用户。

### 返回 422

输入格式不正确、交易字段组合不正确，或市场数据不存在。检查日期、数字、
`transaction_type` 和 symbol。

### 返回 502、503 或 504

通常表示 yfinance 或网络暂时不可用、限流或超时。稍后重试。系统不会因为上游
失败而编造价格。

### 浏览器打不开 `/docs`

确认运行 `make dev` 的终端还开着，并且没有显示红色报错。先访问：

<http://127.0.0.1:8000/health>

正常结果是：

```json
{"status":"ok"}
```

## 13. 哪些文件不要随便改

- `.env`：包含本地秘密；只在确实需要时修改，绝不能分享。
- `uv.lock`：锁定软件版本，不要手工编辑。
- `migrations/`：数据库升级记录，不要删除或改名。
- `compose.yaml`：数据库和缓存配置，不懂时不要修改。
- `pyproject.toml`：项目和依赖配置，不懂时不要修改。

普通使用者通常只需要执行命令，不需要修改代码文件。
